# Inbox Prototype

Question: should `/prototype/inbox` make coworker runs the primary inbox surface by showing blocked runs and all coworker runs together?

Variants:
- `workload`: compact coworker rows with run count, latest run, and a runtime meter.
- `detail`: compact coworker list with selected coworker runtime and run detail panel.
- `modes`: side-by-side inbox/activity sections.
- `heatmap`: coworker rows with small colored runtime strips.
- `sort`: same coworker rows organized by attention, runtime, and latest activity.

Verdict: pending user review.

Design pass: variants were redesigned using the Impeccable product UI guidance: restrained state color, predictable grids, familiar row/table affordances, and separated review intent from runtime analysis.
