# Single Coworker View Prototype

Question: what should the view of a single coworker look like when the user wants to understand status, recent work, context, and next actions without opening the full editor?

Four variants of `/prototype/coworker`, switchable via `?variant=`:

- `A`: Coworker overview. Identity, health, operating model, configuration, and evidence are balanced for quick inspection.
- `B`: Run review. Recent work is the primary object; the coworker is context for a selected run and its outputs.
- `C`: Live desk. Chat and active run status are the primary workspace, with coworker details, tools, and memory in the right rail.
- `D`: Recent runs. A full-page version of the current coworker panel pattern, with compact run rows expanded into reviewable run cards.

Verdict: pending user review.
