# Triage Labels

The skills speak in terms of five canonical triage roles. This repo uses Linear-native labels and statuses, so map each role to the closest appropriate Linear label/status when preparing issue text.

| Canonical role | Linear guidance | Meaning |
| --- | --- | --- |
| `needs-triage` | Choose the Linear label/status that indicates maintainer evaluation is needed | Maintainer needs to evaluate this issue |
| `needs-info` | Choose the Linear label/status that indicates waiting on reporter or missing information | Waiting on reporter for more information |
| `ready-for-agent` | Choose the Linear label/status that indicates the issue is fully specified and AFK-ready | Fully specified, ready for an AFK agent |
| `ready-for-human` | Choose the Linear label/status that indicates human implementation is needed | Requires human implementation |
| `wontfix` | Choose the Linear label/status that indicates the issue will not be actioned | Will not be actioned |

When preparing Linear issue text, include the recommended label/status explicitly. Prefer existing Linear vocabulary over inventing new labels.
