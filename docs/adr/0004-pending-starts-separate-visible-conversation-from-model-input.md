# Pending Starts Separate Visible Conversation From Model Input

Coworkers that require a **Parameter** create a **Pending Start** conversation before any **Generation** exists, so the coworker runner can ask its configured **Parameter Prompt** in the same chat thread the user will later use. We decided that visible coworker-authored question is product conversation history, not model context: when the user replies, the **Generation** starts from the original trigger plus trusted `userInput`, while the prompt snapshot remains available for diagnostics. This preserves a natural chat experience without polluting the model with a pre-run assistant message the model did not actually generate.
